# CRAFT-DF Examples

This directory contains example scripts demonstrating various aspects of the CRAFT-DF deepfake detection system. These examples are designed to help you get started quickly and understand how to use different components of the system.

## Available Examples

### 1. Basic Training (`basic_training.py`)

Demonstrates the simplest way to train a CRAFT-DF model with default settings.

```bash
python examples/basic_training.py
```

**What it shows:**
- Loading default configuration
- Setting up reproducibility
- Initializing the training pipeline
- Running a complete training loop
- Model parameter counting

**Use this when:**
- You want to get started quickly
- You're learning how the training pipeline works
- You want to test the system with default settings

### 2. Custom Configuration (`custom_configuration.py`)

Shows how to create and use custom configurations for different training scenarios.

```bash
python examples/custom_configuration.py
```

**What it shows:**
- Creating custom configuration dictionaries
- Saving configurations to YAML files
- Memory-optimized settings
- Configuration validation
- Comparing different parameter settings

**Use this when:**
- You need to optimize for limited GPU memory
- You want to experiment with different hyperparameters
- You're setting up configurations for different hardware

### 3. Data Preparation (`data_preparation.py`)

Comprehensive demonstration of the data preprocessing pipeline.

```bash
python examples/data_preparation.py
```

**What it shows:**
- Individual component usage (face detection, DWT processing)
- Complete video processing pipeline
- Sample video creation for testing
- Configuration options and their effects
- Output structure and metadata generation

**Use this when:**
- You're preparing your first dataset
- You want to understand the preprocessing steps
- You need to troubleshoot data processing issues
- You're learning about the hierarchical data organization

### 4. Model Inference (`model_inference.py`)

Shows how to load trained models and perform inference on new videos.

```bash
# With your own checkpoint and video
python examples/model_inference.py --checkpoint path/to/model.ckpt --video path/to/video.mp4

# Demo mode (creates sample video)
python examples/model_inference.py --checkpoint path/to/model.ckpt --demo
```

**What it shows:**
- Loading trained model checkpoints
- Single frame inference
- Video-level prediction aggregation
- Confidence scoring and statistics
- Error handling for missing faces

**Use this when:**
- You have a trained model and want to test it
- You need to integrate inference into your application
- You want to understand the prediction pipeline
- You're evaluating model performance on new data

## Configuration Examples

The `configs/` subdirectory contains example configuration files:

### Memory Optimized Configuration

Generated by `custom_configuration.py`, this configuration is optimized for systems with limited GPU memory:

```yaml
model:
  attention_heads: 4        # Fewer attention heads
  attention_dim: 256        # Smaller attention dimension
  freq_dwt_levels: 2        # Reduced DWT levels

training:
  batch_size: 16            # Smaller batch size
  accumulate_grad_batches: 4 # Gradient accumulation
  precision: 16             # Mixed precision
```

## Running the Examples

### Prerequisites

Make sure you have:
1. Installed all dependencies: `pip install -r requirements.txt`
2. Activated your environment: `source craft_df_env/bin/activate`
3. For inference examples: A trained model checkpoint

### Example Workflow

1. **Start with data preparation:**
   ```bash
   python examples/data_preparation.py
   ```

2. **Create custom configuration if needed:**
   ```bash
   python examples/custom_configuration.py
   ```

3. **Run basic training:**
   ```bash
   python examples/basic_training.py
   ```

4. **Test inference with trained model:**
   ```bash
   python examples/model_inference.py --checkpoint checkpoints/best.ckpt --demo
   ```

## Common Use Cases

### Learning the System
Start with `basic_training.py` to understand the overall workflow, then explore `data_preparation.py` to understand preprocessing.

### Memory Constraints
Run `custom_configuration.py` to generate memory-optimized settings, then use the generated configuration file with the main training script.

### Production Deployment
Study `model_inference.py` to understand how to integrate trained models into your applications.

### Debugging Data Issues
Use `data_preparation.py` to test individual components and understand where processing might be failing.

## Troubleshooting Examples

### Common Issues

1. **Import Errors:**
   ```bash
   # Make sure you're in the project root directory
   cd /path/to/craft-df
   python examples/basic_training.py
   ```

2. **Missing Dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

3. **CUDA Issues:**
   ```bash
   # Check GPU availability
   python -c "import torch; print(torch.cuda.is_available())"
   ```

4. **Configuration Errors:**
   ```bash
   # Validate your configuration
   python -c "from craft_df.utils.config import load_config; load_config('your_config.yaml')"
   ```

### Getting Help

If you encounter issues with the examples:

1. Check the main [README.md](../README.md) troubleshooting section
2. Run the examples with Python's verbose flag: `python -v examples/script.py`
3. Check that all file paths are correct and files exist
4. Verify your environment setup and dependencies
